1. Overview & Features 1. Visão Geral & Recursos

Lumina Academy is a premium, enrollment-focused landing page for private schools (K-12). Built with Tailwind CSS v3 CDN, AOS (Animate on Scroll), and vanilla JavaScript.

  • Top Bar — Urgency strip ("Enrollment Open 2026") with ping animation and portal link
  • Off-Canvas Mobile Menu — Slides from right with overlay, ESC close
  • Hero with Image Mask — Organic morphing shape (CSS animation), decorative floating elements
  • Academic Segments — 3 hover-lift cards (Infantil/Fundamental/Médio) with distinct color coding
  • Bento Grid Differentials — Mosaic layout (Bilingual, Robotics, Security, Sports)
  • Animated Counters — Numbers count up with easeOutCubic when scrolled into view
  • Virtual Tour Preview — Wide panoramic placeholder with centered play button
  • News Carousel — Horizontal scroll with prev/next buttons and touch swipe
  • Testimonials — Parent/student quotes with star ratings, 3-column grid
  • Visit Form — Open on page (not modal): Name, Email, Phone (masked), Segment select, with validation
  • Footer — Map, partner logos (Google/Cambridge/LEGO/Apple), quick links
  • AOS Animations — Smooth reveal on scroll for all major sections

3. Color Customization (Tailwind Config) 3. Personalização de Cores (Tailwind Config)

All colors are defined in the tailwind.config object inside index.html (lines 55-100):

tailwind.config = {
  theme: {
    extend: {
      colors: {
        navy: {     // Primary: Trust, professionalism
          50: '#eef2f7',  // Light backgrounds
          900: '#0c2340', // Main brand color
          950: '#061529', // Footer
        },
        mustard: {  // Accent: Creativity, future
          400: '#efc025', // Hover states
          500: '#d4a012', // CTAs, buttons, badges
          600: '#b8830e', // Darker accent
        },
        child: { ... },  // Ed. Infantil (purple)
        fund:  { ... },  // Fundamental (green)
        high:  { ... },  // Ensino Médio (blue)
      }
    }
  }
}

To change the school's brand colors:

  1. Open index.html and find the <script> block with tailwind.config
  2. Replace navy color values with your primary brand color (generate a palette at uicolors.app)
  3. Replace mustard color values with your accent/CTA color
  4. Optionally update the segment colors (child, fund, high) to match your school's palette
  5. Update the SVG icons in the header logo and favicon with your new colors
  6. Update the focus ring color in style.css (search for #d4a012)

Tip: All Tailwind utility classes use these color tokens (e.g., bg-navy-900, text-mustard-500), so changing the config automatically updates the entire page.

Para mudar as cores da escola:

  1. Abra index.html e encontre o bloco <script> com tailwind.config
  2. Substitua os valores de navy pela sua cor primária (gere uma paleta em uicolors.app)
  3. Substitua os valores de mustard pela sua cor de destaque/CTA
  4. Opcionalmente, atualize as cores dos segmentos (child, fund, high) para combinar com a paleta da escola

  6. Atualize a cor do focus ring no style.css (procure por #d4a012)

Dica: Todas as classes utilitárias do Tailwind usam esses tokens de cor (ex: bg-navy-900, text-mustard-500), então mudar a config atualiza automaticamente toda a página.

5. Visit Form 5. Formulário de Visita

The form includes client-side validation:

  • Name — Required, minimum 3 characters
  • Email — Required, validated with regex
  • Phone — Auto-masked as (11) 9 0000-0000, minimum 10 digits
  • Segment — Required, must select an option

On success, the submit button hides and a success message appears with animation. The form currently simulates an API call (1.5s delay). To connect to a real backend:

// In main.js, replace the setTimeout block with:
fetch('/api/visit', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
        name: name.value,
        email: email.value,
        phone: phone.value,
        segment: segment.value
    })
}).then(res => {
    if (res.ok) {
        // Show success
    }
});

O formulário inclui validação client-side:

  • Nome — Obrigatório, mínimo 3 caracteres
  • Email — Obrigatório, validado com regex
  • Telefone — Máscara automática (11) 9 0000-0000, mínimo 10 dígitos
  • Segmento — Obrigatório, deve selecionar uma opção

No sucesso, o botão de envio esconde e uma mensagem de sucesso aparece com animação. O formulário atualmente simula uma chamada de API (1.5s de delay). Para conectar a um backend real, substitua o bloco setTimeout no main.js por um fetch().

6. AOS Animations 6. Animações AOS

The template uses AOS (Animate On Scroll) v2.3.4 via CDN.

Default config in main.js:

AOS.init({
    duration: 700,       // Animation duration (ms)
    easing: 'ease-out-cubic',
    once: true,          // Only animate once
    offset: 60,          // Trigger offset (px)
});

Common attributes used in HTML:

  • data-aos="fade-up" — Fade in from bottom
  • data-aos="fade-right" / "fade-left" — Horizontal fade
  • data-aos="zoom-in" — Scale up
  • data-aos-delay="200" — Stagger delay (ms)
  • data-aos-duration="800" — Override duration

To disable animations, remove data-aos attributes or don't load the AOS library.
