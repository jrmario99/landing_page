Global Color Customization Guide

Learn how to customize the global color palette of your MasterClass Elite landing page by editing the Tailwind CSS configuration.

1. Overview of the Color System

The MasterClass Elite landing page uses a two-tone color system built on Tailwind CSS:

Deep Blue (Primary Background)

Used for backgrounds, cards, and dark UI elements.

Gold (Accent / CTA)

Used for buttons, highlights, icons, and accents.

2. Where Colors Are Defined

Since this project uses the Tailwind CDN, the configuration is defined inline in the <script> tag at the top of index.html:

<script> tailwindcss.config = { theme: { extend: { colors: { deep: { 900: '#0a1128', // Darkest background 800: '#0f1a3e', // Card backgrounds 700: '#162354', // Lighter sections 600: '#1c2d6b', // Hover states 500: '#243b8a', // Lightest deep }, gold: { 100: '#fff7e0', // Lightest gold 200: '#ffe8a3', 300: '#ffd966', // Shimmer highlights 400: '#d4a843', // Icons & borders 500: '#c9952a', // Buttons & CTAs 600: '#a67b1e', // Darkest gold }, }, }, }, } </script>

Note: If you migrate to a build tool (Vite, Webpack, etc.), move this configuration to a tailwind.config.js file with the same structure.

4. How to Change Colors

1

Open index.html

Find the <script> block containing tailwindcss.config near the top of the file.

2

Replace the hex values

Change any hex color code to your desired color. Keep the same shade numbering (lower = darker/lighter depending on the palette).

3

Update the CSS custom properties

Some colors are hard-coded in the <style> block (shimmer gradient, scrollbar, etc.). See Section 5 for details.

4

Save and refresh the browser

Since we use the Tailwind CDN, changes take effect immediately on page load.

5. CSS Custom Properties to Update

Besides the Tailwind config, some colors are used directly in the <style> block. When changing your palette, update these too:

/* Scrollbar thumb color */ ::-webkit-scrollbar-thumb { background: #c9952a; } /* Gold shimmer gradient (text-shimmer class) */ .text-shimmer { background: linear-gradient( 135deg, #ffd966 0%, /* gold-300 */ #c9952a 40%, /* gold-500 */ #ffe8a3 60%, /* gold-200 */ #c9952a 100% /* gold-500 */ ); } /* Play button pulse glow */ @keyframes pulse-ring { 0% { box-shadow: 0 0 0 0 rgba(201,149,42,0.5); } /* gold-500 */ } /* Progress bar gradient */ .progress-bar { background: linear-gradient(90deg, #c9952a, #ffd966); } /* Playlist hover */ .playlist-item:hover { background: linear-gradient(90deg, rgba(201,149,42,0.15), transparent); border-left-color: #c9952a; } /* Scrollbar track (deep-900) */ ::-webkit-scrollbar-track { background: #0a1128; }

Royal Purple & Rose Gold

colors: { deep: { 900: '#120a28', 800: '#1a0f3e', 700: '#261654', 600: '#331c6b', 500: '#42248a', }, gold: { // rename to "rose" optionally 100: '#fff0f0', 200: '#fdd', 300: '#f5b7b1', 400: '#e8967d', 500: '#d4756b', 600: '#b05a4f', }, }

Important: Keep the deep and gold key names unchanged — renaming them would require updating every Tailwind class in the HTML (e.g., bg-deep-900, text-gold-400).

7. Tips & Best Practices

  • Maintain contrast: Ensure your accent color has at least a 4.5:1 contrast ratio against the background for accessibility.
  • Use a color tool: Tools like coolors.co or Tailwind CSS Color Generator help generate harmonious shade scales.
  • Test on mobile: Always preview color changes on mobile devices where screens may render colors differently.
  • Keep shade consistency: The 900 shade should always be the darkest for backgrounds. Higher numbers in the gold scale should be more saturated/darker.
  • Migration path: To use a tailwind.config.js file, install Tailwind via npm, move the config, and replace the CDN script with your build output.

Guia de Personalização de Cores Globais

Aprenda como personalizar a paleta de cores global da sua landing page MasterClass Elite editando a configuração do Tailwind CSS.

1. Visão Geral do Sistema de Cores

A landing page MasterClass Elite usa um sistema de duas cores construído com o Tailwind CSS:

Deep Blue (Fundo Principal)

Usado para fundos, cards e elementos escuros da interface.

Gold (Destaque / CTA)

Usado para botões, destaques, ícones e acentos.

2. Onde as Cores São Definidas

Como este projeto usa o CDN do Tailwind, a configuração é definida inline na tag <script> no topo do arquivo index.html:

<script> tailwindcss.config = { theme: { extend: { colors: { deep: { 900: '#0a1128', // Fundo mais escuro 800: '#0f1a3e', // Fundo de cards 700: '#162354', // Seções mais claras 600: '#1c2d6b', // Estados de hover 500: '#243b8a', // Deep mais claro }, gold: { 100: '#fff7e0', // Gold mais claro 200: '#ffe8a3', 300: '#ffd966', // Destaques shimmer 400: '#d4a843', // Ícones e bordas 500: '#c9952a', // Botões e CTAs 600: '#a67b1e', // Gold mais escuro }, }, }, }, } </script>

Nota: Se você migrar para uma ferramenta de build (Vite, Webpack, etc.), mova esta configuração para um arquivo tailwind.config.js com a mesma estrutura.

4. Como Alterar as Cores

1

Abra o arquivo index.html

Encontre o bloco <script> que contém tailwindcss.config no início do arquivo.

2

Substitua os valores hexadecimais

Altere qualquer código hex para a cor desejada. Mantenha a mesma numeração de tonalidades.

3

Atualize as propriedades CSS

Algumas cores estão diretamente no bloco <style> (gradiente shimmer, scrollbar, etc.). Veja a Seção 5 para detalhes.

4

Salve e atualize o navegador

Como usamos o CDN do Tailwind, as alterações são aplicadas imediatamente ao carregar a página.

5. Propriedades CSS para Atualizar

Além da configuração do Tailwind, algumas cores são usadas diretamente no bloco <style>. Ao mudar sua paleta, atualize estas também:

/* Cor do thumb do scrollbar */ ::-webkit-scrollbar-thumb { background: #c9952a; } /* Gradiente shimmer dourado (classe text-shimmer) */ .text-shimmer { background: linear-gradient( 135deg, #ffd966 0%, /* gold-300 */ #c9952a 40%, /* gold-500 */ #ffe8a3 60%, /* gold-200 */ #c9952a 100% /* gold-500 */ ); } /* Pulso do botão play */ @keyframes pulse-ring { 0% { box-shadow: 0 0 0 0 rgba(201,149,42,0.5); } } /* Gradiente da barra de progresso */ .progress-bar { background: linear-gradient(90deg, #c9952a, #ffd966); } /* Hover da playlist */ .playlist-item:hover { background: linear-gradient(90deg, rgba(201,149,42,0.15), transparent); border-left-color: #c9952a; } /* Track do scrollbar (deep-900) */ ::-webkit-scrollbar-track { background: #0a1128; }

Importante: Mantenha os nomes das chaves deep e gold inalterados — renomeá-los exigiria atualizar todas as classes Tailwind no HTML (ex: bg-deep-900, text-gold-400).

7. Dicas e Boas Práticas

  • Mantenha o contraste: Garanta que sua cor de destaque tenha pelo menos 4.5:1 de contraste contra o fundo para acessibilidade.
  • Use uma ferramenta de cores: Ferramentas como coolors.co ou Tailwind CSS Color Generator ajudam a gerar escalas de tons harmoniosas.
  • Teste no mobile: Sempre pré-visualize as mudanças de cores em dispositivos móveis, onde as telas podem renderizar cores de forma diferente.
  • Mantenha consistência: O tom 900 deve ser sempre o mais escuro para fundos. Números maiores na escala gold devem ser mais saturados/escuros.
  • Caminho de migração: Para usar um arquivo tailwind.config.js, instale o Tailwind via npm, mova a configuração e substitua o script CDN pela saída do build.
