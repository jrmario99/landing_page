01 Overview Visão Geral

What is this template? O que é este template?

SaaS Bento is a modern, dark-themed landing page template built for SaaS products. Its design is inspired by Linear's clean aesthetic — tight typography, glassy cards, and subtle glows on a near-black canvas. SaaS Bento é um template moderno de landing page com tema escuro, criado para produtos SaaS. Seu design é inspirado na estética limpa do Linear — tipografia precisa, cards translúcidos e brilhos sutis sobre um fundo quase preto.

The name comes from its bento-grid features section: a mosaic of cards with varying column and row spans that showcase product capabilities in a visually engaging layout — similar to a Japanese bento box where each compartment holds something different. O nome vem da seção de features em bento-grid: um mosaico de cards com diferentes spans de coluna e linha que exibem as capacidades do produto de forma visualmente envolvente — semelhante a uma marmita japonesa bento, onde cada compartimento tem algo diferente.

No build step Sem build

CDN-only. Open index.html and go. Apenas CDN. Abra o index.html e pronto.

Two pages included Duas páginas incluídas

index.html (landing) + login.html index.html (landing) + login.html

Fully responsive Totalmente responsivo

Mobile-first Tailwind breakpoints throughout. Breakpoints Tailwind mobile-first em todo o projeto.

05 Bento Grid

How the feature mosaic is built Como o mosaico de features é construído

The bento grid is a standard CSS Grid with 12 columns. Cards use Tailwind's col-span-* and row-span-* utilities to create the asymmetric mosaic layout. On mobile the grid collapses to a single column. O bento grid é um CSS Grid padrão com 12 colunas. Os cards usam os utilitários col-span-* e row-span-* do Tailwind para criar o layout mosaico assimétrico. Em mobile o grid colapsa para uma única coluna.

06 3D Dashboard Effect Efeito 3D do Dashboard

Mouse-driven perspective tilt Tilt por perspectiva guiado pelo mouse

The hero dashboard image has a subtle 3D tilt that tracks the user's cursor. It is achieved with pure CSS perspective and rotateX / rotateY transforms, updated by a mousemove listener in main.js. A imagem do dashboard no hero tem um tilt 3D sutil que rastreia o cursor do usuário. É alcançado com perspective e transformações rotateX / rotateY puras em CSS, atualizadas por um listener mousemove em main.js.

HTML structure Estrutura HTML 

<div class="perspective-container">
  <div class="dashboard-tilt">
    <!-- dashboard screenshot / mock-up -->
    <img src="..." alt="Dashboard preview" />
  </div>
</div>

CSS

.perspective-container {
  perspective: 1200px;
}

.dashboard-tilt {
  transform-style: preserve-3d;
  transition: transform 0.1s ease-out;
  will-change: transform;
}

JavaScript (main.js) JavaScript (main.js) 

// Mouse parallax tilt
const container = document.querySelector('.perspective-container');
const tilt     = document.querySelector('.dashboard-tilt');

if (container && tilt) {
  container.addEventListener('mousemove', (e) => {
    const { left, top, width, height } = container.getBoundingClientRect();
    const x = (e.clientX - left) / width  - 0.5;  // -0.5 → 0.5
    const y = (e.clientY - top)  / height - 0.5;

    tilt.style.transform = `rotateY(${x * 12}deg) rotateX(${-y * 8}deg)`;
  });

  container.addEventListener('mouseleave', () => {
    tilt.style.transform = 'rotateY(0deg) rotateX(0deg)';
  });
}

The multipliers * 12 (Y-axis) and * 8 (X-axis) cap the maximum rotation angles. Adjust them to taste — lower values give a subtler effect, higher values make it more dramatic. Os multiplicadores * 12 (eixo Y) e * 8 (eixo X) limitam os ângulos máximos de rotação. Ajuste-os ao gosto — valores menores dão um efeito mais sutil, valores maiores tornam-no mais dramático.

07 Scroll Animations Animações de Scroll

IntersectionObserver-based fade-in Fade-in baseado em IntersectionObserver

Elements with the class .fade-in-section start invisible and slightly translated downward. When they enter the viewport, .is-visible is added, triggering a CSS transition to full opacity and zero translation. Elementos com a classe .fade-in-section começam invisíveis e ligeiramente transladados para baixo. Quando entram na viewport, .is-visible é adicionado, ativando uma transição CSS para opacidade total e translação zero.

CSS

.fade-in-section {
  opacity: 0;
  transform: translateY(24px);
  transition: opacity 0.6s ease, transform 0.6s ease;
}

.fade-in-section.is-visible {
  opacity: 1;
  transform: translateY(0);
}

/* Staggered delay utilities */
.delay-100 { transition-delay: 100ms; }
.delay-200 { transition-delay: 200ms; }
.delay-300 { transition-delay: 300ms; }
.delay-400 { transition-delay: 400ms; }

JavaScript (main.js) JavaScript (main.js) 

const observer = new IntersectionObserver(
  (entries) => {
    entries.forEach((entry) => {
      if (entry.isIntersecting) {
        entry.target.classList.add('is-visible');
        observer.unobserve(entry.target); // fire once
      }
    });
  },
  { threshold: 0.15 }
);

document
  .querySelectorAll('.fade-in-section')
  .forEach((el) => observer.observe(el));

Using observer.unobserve() after the first intersection means each element animates in only once — subsequent scrolls back up won't re-trigger the animation. Change threshold: 0.15 to control how much of the element must be visible before it triggers. Usar observer.unobserve() após a primeira interseção significa que cada elemento anima apenas uma vez — rolar de volta para cima não reativa a animação. Altere threshold: 0.15 para controlar quanto do elemento deve estar visível antes de disparar.

Autofill background fix Correção do background de autofill

Browsers inject a bright yellow or white background on autofilled inputs. Overriding it requires a CSS trick using a long transition-delay on background-color: Navegadores injetam um fundo amarelo brilhante ou branco em inputs preenchidos automaticamente. Sobrescrevê-lo requer um truque CSS usando um longo transition-delay em background-color: 

input:-webkit-autofill,
input:-webkit-autofill:hover,
input:-webkit-autofill:focus {
  -webkit-box-shadow: 0 0 0 1000px #131418 inset;
  -webkit-text-fill-color: #ffffff;
  transition: background-color 9999s ease-in-out 0s;
}

09 Pricing Section Seção de Preços

Alpine.js toggle and three pricing tiers Toggle Alpine.js e três planos de preço

The pricing section is wrapped in an Alpine.js component (x-data) that holds a single boolean: annual. A toggle button switches between monthly and annual billing; prices update instantly via x-show / x-text. A seção de preços é envolta em um componente Alpine.js (x-data) que mantém um único booleano: annual. Um botão toggle alterna entre cobrança mensal e anual; os preços atualizam instantaneamente via x-show / x-text.

Alpine.js component Componente Alpine.js 

<section x-data="{ annual: true }">

  <!-- Toggle -->
  <button @click="annual = !annual">
    <span :class="annual ? 'active' : ''">Annual</span>
    <span :class="!annual ? 'active' : ''">Monthly</span>
  </button>

  <!-- Annual price (shown when annual === true) -->
  <span x-show="annual">$8</span>
  <span x-show="!annual">$12</span>

</section>
