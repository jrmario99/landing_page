1. Overview & Features 1. Visão Geral & Recursos

PrevLaw is an accessible, mobile-first landing page designed for social security lawyers (advogados previdenciários) in Brazil. Built with Tailwind CSS v3 and vanilla JavaScript.

  • Accessibility Widget — Font resize (A-/A/A+) and high contrast toggle, preferences saved in localStorage
  • Eligibility Quiz — 6 interactive options that redirect to WhatsApp with pre-filled situation message
  • Testimonials Carousel — Touch-swipe support, keyboard navigation, autoplay with pause-on-hover
  • FAQ Accordion — Close-others-on-open behavior, ARIA attributes
  • Sticky Mobile WhatsApp Bar — Appears after 400px scroll, hidden on desktop
  • 48px minimum touch targets — All interactive elements meet accessibility standards
  • SEO optimized — Schema.org Attorney markup, Open Graph tags
  • Scroll reveal animations — Staggered entrance for practice area cards

PrevLaw é uma landing page acessível e mobile-first projetada para advogados previdenciários no Brasil. Construída com Tailwind CSS v3 e JavaScript puro.

  • Widget de Acessibilidade — Redimensionamento de fonte (A-/A/A+) e modo alto contraste, preferências salvas no localStorage
  • Quiz de Elegibilidade — 6 opções interativas que redirecionam para o WhatsApp com mensagem pré-preenchida
  • Carrossel de Depoimentos — Suporte a toque/swipe, navegação por teclado, autoplay com pausa ao hover
  • FAQ em Acordeão — Fecha outros ao abrir, atributos ARIA
  • Barra Fixa Mobile de WhatsApp — Aparece após 400px de scroll, escondida no desktop
  • Mínimo de 48px para alvos de toque — Todos os elementos interativos atendem padrões de acessibilidade
  • SEO otimizado — Schema.org tipo Attorney, tags Open Graph
  • Animações de scroll reveal — Entrada escalonada para cards de áreas de atuação

3. Quick Setup 3. Configuração Rápida

  1. Replace images in assets/img/ with your own photos
  2. Search for 5511980000000 in both index.html and main.js and replace with your actual WhatsApp number
  3. Update the lawyer's name, OAB number, address, and contact info (search for <!-- EDITABLE comments)
  4. Replace the Google Maps iframe with your actual embed code
  5. Deploy to any static hosting (Netlify, Vercel, GitHub Pages, etc.)

No build step required! Uses Tailwind CSS v3 CDN — just upload and go.

4. WhatsApp Configuration 4. Configuração do WhatsApp

WhatsApp links appear in 5 places:

  1. Desktop navbar CTA button
  2. Mobile menu CTA button
  3. Hero section CTA button
  4. Sticky mobile footer bar
  5. Quiz redirect (in main.js)

All locations in index.html are marked with <!-- EDITABLE: WhatsApp link --> comments.

In main.js, search for whatsappNumber variable (around line 128) to change the quiz redirect number.

Format: Country code + number, no spaces or dashes. Example: 5511999999999

5. Accessibility Widget 5. Widget de Acessibilidade

The top bar includes 4 accessibility controls:

A-Decrease font size (min 87.5%)html.font-sm

Preferences are saved in localStorage under keys prevlaw-font-step and prevlaw-contrast.

Font sizes scale via the <html> root element's font-size (rem-based), so all Tailwind text utilities scale proportionally.

6. Color Customization 6. Personalização de Cores

Colors are defined in the Tailwind config inside index.html <script> tag (lines 48–95):

  • navy-* — Primary palette (dark blue shades, text, backgrounds)
  • warm-* — Accent palette (warm orange tones, CTAs, highlights)
  • gold-* — Decorative accents (shield icons, badges)
  • green-600/700 — WhatsApp buttons (Tailwind default greens)

To change the color scheme, edit the tailwind.config object in the HTML head. All classes throughout the page use these custom colors.

Also update the high-contrast colors in style.css section 3 if you change the accent colors.

As cores são definidas na config do Tailwind dentro do index.html na tag <script> (linhas 48–95):

  • navy-* — Paleta primária (tons de azul escuro, textos, fundos)
  • warm-* — Paleta de destaque (tons laranja quente, CTAs, destaques)
  • gold-* — Acentos decorativos (ícones de escudo, badges)
  • green-600/700 — Botões WhatsApp (verdes padrão do Tailwind)

Para mudar o esquema de cores, edite o objeto tailwind.config no head do HTML. Todas as classes na página usam essas cores customizadas.

Atualize também as cores de alto contraste na seção 3 do style.css se você alterar as cores de destaque.

7. Sections Guide 7. Guia de Seções

Accessibility Bar

Top bar with font resize buttons (A-/A/A+) and high contrast toggle. Also shows phone and email on desktop.

Navbar

Sticky navigation with logo, desktop links, WhatsApp CTA, and hamburger mobile menu. Adds shadow on scroll.

Hero

Two-column layout with headline, CTA buttons (WhatsApp + Phone), trust badges, and lawyer photo with OAB badge overlay.

Dark background section with 6 clickable options. Each redirects to WhatsApp with the selected situation pre-filled in the message.

Two-column layout with banner image and lawyer bio. Includes 4 stat boxes (clients, success rate, years, coverage).

Full-width carousel with 4 slides. Prev/Next buttons, dot indicators, autoplay (6s), swipe support, keyboard arrows.

FAQ

5 accordion items. Opening one closes others. ARIA expanded states are managed by JS.

Footer

4-column grid: brand info, navigation links, contact details, Google Maps embed.

Sticky Mobile Bar

Fixed bottom bar (mobile only) with WhatsApp CTA. Hidden on desktop (lg: breakpoint). Appears after 400px scroll.

9. SEO & Meta Tags 9. SEO & Meta Tags

The template includes:

  • Schema.org — @type: Attorney structured data with name, address, phone, URL
  • Open Graph — og:title, og:description, og:image for social sharing
  • Meta tags — description, keywords, robots (index, follow)
  • Semantic HTML — proper heading hierarchy (h1 → h2 → h3), sections, nav, footer, blockquote
  • Alt text — All images include descriptive alt attributes
  • Lazy loading — Images below the fold use loading="lazy"

Update the Schema.org JSON-LD block (lines 19–38 in index.html) with your actual business info.

O template inclui:

  • Schema.org — Dados estruturados @type: Attorney com nome, endereço, telefone, URL
  • Open Graph — og:title, og:description, og:image para compartilhamento social
  • Meta tags — description, keywords, robots (index, follow)
  • HTML Semântico — hierarquia correta de headings (h1 → h2 → h3), sections, nav, footer, blockquote
  • Texto alternativo — Todas as imagens possuem atributos alt descritivos
  • Lazy loading — Imagens abaixo da dobra usam loading="lazy"

Atualize o bloco JSON-LD do Schema.org (linhas 19–38 no index.html) com as informações reais do seu negócio.
